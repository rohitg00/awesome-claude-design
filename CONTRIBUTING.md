# Contributing

One bar: **would a working designer or engineer find this useful in the next 48 hours?**

## How to submit

1. Fork the repo
2. Add your entry to the right section in `README.md`, or drop a file in the matching folder (`design-md/`, `prompts/`, `recipes/`, `showcase/`)
3. Open a PR — title format: `[category] name — one-line value`

## What fits

### DESIGN.md files (`/design-md/<aesthetic-family>/`)
- Pick an aesthetic family: `editorial`, `terminal`, `warm`, `data-dense`, `cinematic`, `playful`, `glass`, `brutalist`, `indie`, or `remix`
- Indie brands, OSS projects, personal portfolios, local businesses, remix combos
- Must include all 9 canonical sections (see README → DESIGN.md by Aesthetic Family)
- Include swatch (hex values), type spec, and 1200x630 preview PNG
- Do not duplicate VoltAgent entries — link upstream instead
- Remix recipes welcome: `/design-md/remix/<brand-a>-x-<brand-b>.md`

### Prompt packs (`/prompts`)
- One markdown file per prompt
- Required: prompt text, expected input, expected output, before/after screenshot
- Reproducibility > cleverness

### Recipes (`/recipes`)
- End-to-end workflows you actually ran
- Steps with exact commands/clicks
- Screenshots of each stage
- Honest notes on what broke

### Showcase (`/showcase`)
- Real projects shipped with Claude Design
- Required: live URL, DESIGN.md used, 2–3 screenshots, one-paragraph story

### Video teardowns
- Minimum 2K views OR substantive critique (not pure promotion)
- Note the angle (walkthrough, head-to-head, skeptical review, etc.)

## What gets rejected

- Affiliate links
- Screenshots without reproducible prompts
- "Look at this cool thing!" with no context
- Slop — generic outputs, recycled prompts, no judgment applied
- Emoji-heavy commits or headers
- Anything that requires trusting the submitter's word (show the receipt)

## Style

- No emojis in README or file headers
- Real quotes with source + engagement numbers when citing community
- Link to primary sources (Anthropic blog, creator videos, threads) not aggregators
- Active voice. Short lines.

## Code of conduct

Don't be a jerk. Critiques are welcome — personal attacks are not.
